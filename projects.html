LC

This is the project I spend most of my time these days.
It is the compiler I build for my PhD and I used as a research tool for some years now.
It is not intended to be used in production and must instead be considered as a buggy(-woogie) but unique tool I use, with a lot of fun, to explore the JIT compilation world.
LC is an optimizing JIT compiler for Scheme written in Scheme using new JIT compilation techniques presented in some research papers.
LC officially stands for Lazy Compiler but also for Le Compilateur that means The Compiler in French.
Unfortunately, I spend less time on it since the beginning of 2019 because I'm focusing on writing my PhD dissertation.

Timbit

Timbit is an attempt to build a new Scheme implementation from scratch that secretely wants to be the successor of LC.
For now, Timbit is slow because it is an interpreter and is not optimized.
It is written in C++ and uses an intermediate representation that is designed for the purpose of becoming a generic bytecode for dynamic languages.
Unlike LC, it features a REPL and is intended to be used for real programs instead of being a research tool.
Timbit follows the Scheme implementation names tradition (like the Orbit, Rabbit and Gambit compilers).
It is named for these small sweet balls.
